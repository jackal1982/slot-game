## 遊戲機制說明

### Free Spins 觸發
- **條件**：軸1~3 出現 3 個 Scatter
- **獎勵**：10 局 Free Spins
- **倍率**：1×（M1 隨機百搭特色補強）

### Retrigger
- Free Game 中再出現 3 SC：+10 局，上限 50 局

### M1 隨機百搭特色（Free Game 專用）
- **觸發條件**：M1 在任意 Way 有 3+ 軸連線
- **效果**：在軸2~5 隨機位置放置額外 Wild（跳過已有 Wild 的格子）
- **Wild 數量分佈**：
  - 2~4 個：60%
  - 5~8 個：30%
  - 9~12 個：8%
  - 13~16 個：2%
- **重新計算**：放置後重新評估所有 Ways 贏分

## 設計原則

### Base Game
1. **軸1**：無 Wild，SC 觸發源，以低值符號為主
2. **軸2~3**：同時有 SC 和 Wild，提供中值符號組合
3. **軸4~5**：較多 Wild，強化高值符號連線
4. **稀有高值符號**：M2/M3 在各軸僅 3~4 個，保持中等波動

### Free Game
1. **M1 密度提升**：每軸 6 個（vs Base M2+M3 共 6~7 個），觸發隨機百搭特色
2. **Wild 大幅增加**：軸2~5 Wild 分別 6/7/8/9 個，明顯高於 Base 的 4/5/5/6 個
3. **SC 增加**：各軸 3 個，Retrigger 觸發率較高

```javascript
// 對每個付費符號 sym（非 SC、非 WD），計算每軸可見4格中 sym 或 WD 的數量
// 從左至右連續非零軸相乘 = ways
// win += ways × payrate[sym][len]  （len = 連續匹配軸數）

// WD 本身在軸1無 WD，故無獨立 WD Ways win（避免雙重計算）
```
